Introduction

This project was created by Ankur Shukla and Harshul Jain, students of Netaji Subhas University of Technology, as participants of the IEEE PEC Hackathon 2020. The theme chosen for the competetion was 'Resume Filtering'. The details of the theme and the problem statement can be found here. We chose this theme mainly because it was quite relatable to our current scenerio. There are millions of students applying for jobs every year, so we thought such service would of great use. Technology Stack

This section thoroughly explains the technologies used and the reason for choosing them. All the technologies used are open source.

  1. Python (Programming Language):
    2. There is no denying that Python is so far the best language to be used for machine learning tasks, and since this is majorly a machine learning problem, we used Python as the main language. Python is also very commonly used as a web development language. Frameworks like Django and Flask are based on Python. In this project we used Flask as the backend framework, explaination of which is provided later.

  2. Flask (Web Framework):
    3. Flask is a micro web framework written in Python. It is classified as a microframework because it does not require particular tools or libraries. It has no database abstraction layer, form validation, or any other components where pre-existing third-party libraries provide common functions. However, Flask supports extensions that can add application features as if they were implemented in Flask itself. Extensions exist for object-relational mappers, form validation, upload handling, various open authentication technologies and several common framework related tools. Extensions are updated far more frequently than the core Flask program. ~Wikipedia

    The major reason for using flask was its lightweight nature. We had to build a prototype that is easy to code yet functions good. Flask matches all the criteria we needed in a web framework. Flask makes backend deployment really easy and fast. Its powerful enough to be used in live apps. Companies like Pinterest and LinkedIn use Flask as a backend for their websites.

  3. Heroku (Deployment):
    4. Heroku is a cloud platform as a service (PaaS) supporting several programming languages. One of the first cloud platforms, Heroku has been in development since June 2007, when it supported only the Ruby programming language, but now supports Java, Node.js, Scala, Clojure, Python, PHP, and Go. For this reason, Heroku is said to be a polyglot platform as it has features for a developer to build, run and scale applications in a similar manner across most languages. ~Wikipedia

    We preferred Heroku beacuse it is most commonly used Open Source platform for deploying web apps. It offers very fast deployment methods and easy error logging. Heroku offers upto 500 megabytes of deplyment storage in free accounts.

  4. Bootstrap (CSS):
    5. Boootstrap offers really quick and beautiful methods for creating front-end of web apps.

  5. Keras (Machine Learning Framework):
    6. Keras is an open-source neural-network library written in Python. It is capable of running on top of TensorFlow, Microsoft Cognitive Toolkit, R, Theano, or PlaidML. Designed to enable fast experimentation with deep neural networks, it focuses on being user-friendly, modular, and extensible. It was developed as part of the research effort of project ONEIROS (Open-ended Neuro-Electronic Intelligent Robot Operating System). ~Wikipedia

    Its lightweight nature was the major motivation for choosing this framework. We wanted to deploy our model to web and to do so we needed a framework that does not consumes much server space, since as mentioned earlier, Heroku offers only 500 megabytes of storage space. There were two other options - Tensorflow and PyTorch, but due to their large size, they were discarded. Also it was not wise to waste space on framework components that are of no use to the app. The following image shows the model architecture.



  6. PyPDF2 (Parse pdf File):
    7. PyPDF2 is used to work with pdf files. It is used to read the document , split pdfs page by page, extract document information and more.

  7. NumPy:
    8. NumPy is a library for the Python programming language, adding support for large, multi-dimensional arrays and matrices, along with a large collection of high-level mathematical functions to operate on these arrays. The ancestor of NumPy, Numeric, was originally created by Jim Hugunin with contributions from several other developers. In 2005, Travis Oliphant created NumPy by incorporating features of the competing Numarray into Numeric, with extensive modifications. NumPy is open-source software and has many contributors. ~Wikipedia

  8. NLTK (Language Processing):
    9. NLTK library is used to preprocess the text before feeding it into the neural network. It has a pretty good collection of stop words and which removes redundant text from the input.

  9. Pandas (Reading datasets):
    10. We have used Pandas library to read csv files which contained the training data for the application.
